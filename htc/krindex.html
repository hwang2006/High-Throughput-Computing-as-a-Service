HTCaaS

April 23, 2013, Main Page

높은 처리량 컴퓨팅 (HTC)는 독립적 많은 느슨하게 결합 된 작업을 실행하는 구성 (이 그들 사이에 필요한 통신이 없음) 그러나 시간이 상대적으로 오랜 기간 동안 계산 많은 양의 전력을 필요로합니다. 이러한 콘도르 또는 BOINC 같은 미들웨어 시스템은 성공적으로 활용 컴퓨팅 자원의 다수의 엄청난 컴퓨팅 파워를 달성했다. 그러나 작업 및 과학 응용 프로그램 증가의 복잡성 숫자로, 그 안에서 주어진 과학적 문제를 해결하기 위해 자원 일반적으로 하나의 유형 (예를 들어, 워크 스테이션의 클러스터, 데스크탑 컴퓨터 인터넷을 통해) 채용 기존의 미들웨어 시스템을위한 도전이됩니다 시간의 합리적인 금액입니다. 또한, 작업 실행 시간 당 상대적으로 짧은으로 처리 할 수​​백만 또는 작업 수십억을 요구하는 최근의 새로운 응용 프로그램은 많은 태스크 컴퓨팅 (MTC)로 확장하는 기존의 HTC를 주도했다.

따라서 효율적으로 복잡하고 까다로운 과학 애플리케이션을 지원하기 위해서는 슈퍼 컴퓨터 그리드, 심지어 클라우드를 포함하여 가능한 한 많은 컴퓨팅 자원을 활용하기 위해 불가피하다. 그러나, 연구자들이 효과적으로 작업의 수 (제출되어야하는 독립적 인 리소스 공급자에 의해 제어되는 사용 가능한 자원 활용을 위해 도전 한 번 ) (파라미터 스윕 또는 N-바디 계산)을 극적으로 증가합니다.

우리는 설계 및 구현 HTCaaS을 (서비스로 높은 처리량 컴퓨팅) 사용자의 다양한 컴퓨팅 리소스 활용의 이질성과 복잡성을 숨길 수있는 시스템을 효율적으로 효과적으로 사용할 수있는 모든 컴퓨팅 자원을 관리 및 활용하여 한 번에 작업의 다수를 제출 .


우리의 디자인 철학은 다음과 같다 :

  • 사용의 용이성 : 우리는 작업 및 컴퓨팅 자원의 대량 처리에 대한 사용자의 오버 헤드를 최소화
  • 지능형 리소스 선택 : HTCaaS가 자동으로 더 반응하고 효과적인 리소스를 선택하고 동적으로 획득 한 자원을 조정하여 부하 전류에 적응할 수
  • 리소스에 대한 플러그 인터페이스 : 우리는 하드 코딩하지 않고 이기종 컴퓨팅 리소스에 액세스 GANGA의 플러그인기구를 채용
  • 많은 클라이언트 인터페이스 지원 : 클라이언트 인터페이스의 넓은 범위가 기본 WS 인터페이스, 자바 API 및 클라이언트 도구 (CLI, GUI)를 포함한 지원

시스템 아키텍처 및 구성 요소

HTCaaS 시스템은 다섯 개의 서버 측 모듈 (계정 관리, 사용자 데이터 관리자, 작업 관리자, 에이전트 관리자, 모니터링 관리자)와 두 개의 클라이언트 측 도구 (명령 줄 인터페이스와 그래픽 사용자 인터페이스)로 구성되어 있습니다.



우리의 시스템에서 작업을 수행 할 연산을 설명하는 데이터 및 관련 프로필입니다. 사용자가 매개 변수 스위프 또는 N-바디 계산을 채용하여 작업의 다수를 제출 할 수 있기 때문에, HTCaaS는의 개념을 도입 메타 작업 OGF JSDL 표준을 기반으로 높은 수준의 작업 설명을 지정합니다. 메타 작업이 제출되면, HTCaaS는 자동으로 작업 관리자가 관리하는 작업 큐 (ActiveMQ에서 구현)로 여러 작업을 삽입 그들로 그것을 분할. 필요한 입력 데이터와 생산 결과의 모든 사용자 데이터 관리자에 저장됩니다. 작업은 우리의 시스템에 제출되면, 에이전트는 (자바로 구현) 에이전트 관리자 및 슈퍼 컴퓨터 그리드, 그리고 구름에서 공정 작업에서 파견된다. HTCaaS은 에이전트 기반 고용 파견 멀티 레벨 스케줄링 및 능률적 인 작업을 하므로 일괄 작업으로 에이전트를 제출하여 배치 스케줄러에 대한 첫 번째 수준 요청 (예를 들어, PLSI 슈퍼 컴퓨터의로드 레벨러, 그리드 gLite, 아마존 EC2를위한 PBS) 보유 자원을이 다음 각 에이전트는 사전 메커니즘을 파견 가볍고 빠른 작업을 구현하는 작업 관리자에서 작업을 가져옵니다.

따라서 HTCaaS의 사용자는 최종 결과를 제출하고 실행 수천 개의 일자리 (이 단순히 하나의 JSDL 스크립트에 의해 표현 될 수있다) 수백 자동화 된 프로세스 내에서 효과적으로 모니터링하고 처리 할 수​​ 있습니다. 스크립트의 XML 스타일에 익숙하지 않은 사람들을 위해, 우리는 또한 자동으로 우리의 시스템에 제출 될 수 있도록 사용자의 입력에 따라 JSDL 스크립트를 생성 할 수 있습니다 사용하기 쉬운 GUI 도구를 제공합니다. HTCaaS 시스템에서 작업 제출 및 실행의 전반적인 단계는 다음과 같다 :



1. 사용자 로그인의 HTCaaS 및 사용자 데이터 관리자를 통해 이미지 입력​​ 데이터를.
2. 사용자가 여러 개의 작업으로 구성 될 수 있습니다 메타 작업 (JSDL로 작성) 제출합니다.
3. HTCaaS 자동으로 사양과 작업 큐에 삽입에 따라 여러 작업으로 메타 작업을 나눕니다.
4. 작업 요구 사항과 리소스 가용성에 따라 에이전트 관리자를 전달 에이전트.
